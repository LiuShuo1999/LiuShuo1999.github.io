Scoring Formulas

These notes explain what each dimension means and how the visible scores are computed. The weights and caps can be changed in Advanced filters.

Relation Footprint

Measures how many PRs, issues, and commits are directly related to the selected issue.

pull_request_count * w_pr + issue_count * w_issue + commit_count * w_commit
关联PR数量 * PR权重 + 关联Issue数量 * Issue权重 + 关联Commit数量 * Commit权重

Discussion Intensity

Measures discussion strength from comment volume, unique participants, maintainer participation, and real back-and-forth exchanges.

comment_count * w_comment + commenter_count * w_commenter + maintainer_participant_count * w_maintainer + back_and_forth_turns * w_turn
评论数 * 评论数权重 + 独立评论人数 * 独立评论人数权重 + 维护者参与人数 * 维护者参与权重 + 来回讨论轮次 * 来回讨论权重

back_and_forth_turns is computed from comments sorted by time: each adjacent author change adds one turn. For example, A, A, B, B, A gives 2 turns.

Temporal Persistence

Measures whether an issue keeps attracting attention over time, including long active span, cross-year/month activity, revival after silence, and recent activity.

min(active_span_days / 365, cap_active_span_years) * w_active_span + active_year_count * w_active_year + active_month_count * w_active_month + revival_180_count * w_revival_180 + revival_365_count * w_revival_365 + recent_activity_bonus * w_recent_activity
min(活跃跨度天数 / 365, 活跃跨度年份上限) * 活跃跨度权重 + 活跃年份数 * 活跃年份权重 + 活跃月份数 * 活跃月份权重 + 沉寂超过180天后再次活跃次数 * 180天复活权重 + 沉寂超过365天后再次活跃次数 * 365天复活权重 + 近期活跃奖励 * 近期活跃权重

active_span_days is last activity time minus issue creation time, not necessarily open time to today. recent_activity_bonus defaults to 1 only when the issue was active within the last 90 days.

Active Open Pain

Measures long-standing open issues that may still represent unresolved pain. It only applies to open issues older than the configured minimum age.

min(issue_age_days / 365, cap_issue_age_years) * w_issue_age * recency + min(comment_count, cap_comment_count) * w_comment
min(issue存在天数 / 365, issue年龄年份上限) * issue年龄权重 * 近期活跃系数 + min(评论数, 评论数上限) * 评论数权重
days_since_last_activity <= hot_days: recency = recency_hot days_since_last_activity <= warm_days: recency = recency_warm days_since_last_activity > warm_days: recency = recency_cold
距离最后活跃天数 <= 热活跃天数阈值：近期活跃系数 = 热活跃系数 距离最后活跃天数 <= 温活跃天数阈值：近期活跃系数 = 温活跃系数 距离最后活跃天数 > 温活跃天数阈值：近期活跃系数 = 冷活跃系数

Historical Anchor

Measures whether a closed issue continues to serve as a historical root cause, design decision, or repair entry point.

post_close_reference_count * w_post_close_reference + post_close_reference_year_count * w_post_close_year
关闭后引用次数 * 关闭后引用权重 + 关闭后被引用年份数 * 关闭后引用年份权重

Recurrence Signal

Measures whether a problem appears repeatedly or whether a previous fix may have been incomplete.

min(reopened_count, cap_reopened_count) * w_reopened
min(重新打开次数, 重新打开次数上限) * 重新打开权重
